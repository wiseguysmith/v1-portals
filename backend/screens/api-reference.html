API Reference
v1.0 · Draft spec
ℹ
This is the v1 API contract for backend developers. All endpoints, schemas, and webhook events described here represent the target backend implementation. Adapters are designed to be provider-swappable — see the adapter pattern notes in each section.

MethodPathDescription
GET/api/v1/dealsList all deals accessible to the authenticated role
GET/api/v1/deals/:idGet full deal detail including gate status and checklist

Adapter Pattern Note
ℹ
All vendor integrations (KYC, custody, signing, chain RPC) use a provider-agnostic adapter layer. Swapping from Parallel Markets to another KYC provider, or from Base to another chain, requires changing the adapter config only — no application code changes. Adapters implement a standard interface per vendor category. See /src/adapters/ in the backend repo.

Stack Notes
ℹ
Database: PostgreSQL with row-level security per deal_scope.
Queue: Redis + BullMQ for background jobs (KYC polling, gate evaluation, email).
File storage: S3 presigned upload URLs — never proxy file bytes through the API server.
Token deployment: ERC-3643 on Base. Use hardhat or foundry for contract management. Keep deployer key in AWS Secrets Manager or Vault — never in .env for production.
Adapters: All vendor calls go through /src/adapters/{category}/{provider}.ts — swap provider by changing config, not code.
Auth: Magic link email login via SendGrid. SSO via OAuth 2.0 (optional). All portal routes are server-rendered and gated by JWT middleware.
